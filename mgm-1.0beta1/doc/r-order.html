Resource: order

applicability

Module class only

possible settings

floating point value or values in comma delimited list (every module has different default)

description

order is used to specify where to place a given module class in the left-right or top-bottom order of module appearance. Some modules may be used more than once; these modules may take a comma seperated list of positions (see module documentation to determine which modules may have multiple instances).

Module ordering is determined by making a list of the module's order values (perhas making multiple entries for a module), then sorting the list (and thus modules) in ascending order. Module documentation lists the default order settings of each module.
