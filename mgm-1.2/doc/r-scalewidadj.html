Resource: scalewidadj

applicability

Toplevel, Module class or instance

possible settings

0<=n
(default varies my module, global default is 100)

description

'scalewidadj' sets the proportional width that the module requests over the minimum width requested by minx or miny (depends on stacking). When bars and stack differ (eg, vertical bars and horizontal stacking, the typical case), the scalewidadj values for all the active modules are summed and each module is allotted a width proportional to its scalewidadj value plus it's minimum size value (see the geometry page for a detailed discussion of space allocation).

When the graph bars and stacking are in the same direction, the maximum requested width of any module is used only to determine main window size and each module is placed within the same (potentially excess) width according to its scalejustify setting.
